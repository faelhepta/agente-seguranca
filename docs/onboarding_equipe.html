Passo 1 de 3

Pre-requisitos

Instale isso uma unica vez em cada maquina

▶

Node.js 18 ou superior

Necessario para rodar o Claude Code

node --version

Se nao tiver: baixe em nodejs.org

◆

Claude Code

O motor de analise. Instalar via npm:

npm install -g @anthropic-ai/claude-code

Verificar: claude --version

🔑

Autenticacao

Fazer login com a conta da equipe:

claude auth login

Solicitar a chave de API ao responsavel

📄

Git (opcional)

Para clonar o repositorio. Alternativa: baixar o ZIP direto do GitHub.

git --version
Passo 2 de 3

Instalacao

Feita uma unica vez por maquina

1

Clonar ou baixar o repositorio

Via Git:

git clone https://github.com/faelhepta/agente-seguranca.git

Ou baixar o ZIP em: github.com/faelhepta/agente-seguranca → botao Code → Download ZIP

2

Abrir o PowerShell e liberar execucao de scripts

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Necessario apenas uma vez por maquina.

3

Entrar na pasta e rodar o instalador

cd agente-seguranca .\instalar.ps1

O instalador copia tudo para os lugares certos automaticamente.

Passo 2 de 3 - Detalhes

O que o instalador coloca na maquina

Scripts de analise

Pasta: Documentos\Seguranca-TI\

  • analisar.ps1 — script principal
  • rodar_analise.ps1 — modo avancado
  • Checklists e runbook
  • Apresentacao do time

Skills do Claude Code

Pasta: .claude\skills\

  • /relatorio-fabrica — gera slides HTML
  • /reanalise-fabrica — verifica correcoes

Template de projetos

Pasta: Documentos\projetos\_TEMPLATE\

  • codigo/
  • documentacao/
  • infraestrutura/
  • relatorios/ e apresentacoes/

Resultado esperado

INSTALACAO CONCLUIDA COM SUCESSO

Se aparecer esse texto, esta tudo pronto.

Passo 3 de 3

Rodando a analise

Fazer isso para cada projeto recebido

1

Organizar os arquivos do projeto

Copie o template e coloque os artefatos da fabrica nas subpastas:

Documentos\projetos\NomeDoProjeto\ codigo\ <- codigo-fonte documentacao\ <- PDFs, diagramas, especificacoes infraestrutura\ <- diagramas de rede, Terraform...
2

Rodar o script de analise

cd "$env:USERPROFILE\Documents\Seguranca-TI" .\analisar.ps1

O script vai pedir o caminho da pasta do projeto e rodar tudo automaticamente.

3

Aguardar e revisar

O processo leva entre 1 e 2 horas. Ao final, abrir os relatorios e as apresentacoes HTML geradas na pasta apresentacoes\ do projeto.

Referencia

Onde ficam os arquivos

Repositorio (agente-seguranca)

agente-seguranca\ instalar.ps1 docs\ analisar.ps1 <- script principal rodar_analise.ps1 checklists e docs skills\ relatorio-fabrica.md reanalise-fabrica.md

Apos instalacao (maquina do analista)

Documentos\ Seguranca-TI\ analisar.ps1 <- rodar aqui rodar_analise.ps1 checklists e docs projetos\ _TEMPLATE\ NomeDoProjeto\ codigo\ relatorios\ apresentacoes\
A pasta agente-seguranca (repositorio) pode ficar em qualquer lugar. Depois de instalar, o dia a dia e feito a partir de Documentos\Seguranca-TI\.
Manutencao

Atualizando a equipe

Quando houver novidades nos scripts ou documentos

1

Responsavel faz a alteracao e publica no GitHub

git add . git commit -m "descricao da mudanca" git push
2

Cada membro da equipe atualiza sua maquina

cd agente-seguranca git pull .\instalar.ps1 -Modo update

O modo update so atualiza os arquivos, sem recriar a estrutura de projetos.
